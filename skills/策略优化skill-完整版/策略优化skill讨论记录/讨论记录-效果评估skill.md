# 效果评估 Skill — 讨论记录

## 一、核心定位

将原始明细数据加工成策略库格式的效果数据。本质是一个**效果计算工具**——策略库需要每个策略的完整效果数据，但原始数据不会直接给出，需要通过某种方法计算。

## 二、计算范围

按照业务背景对齐中定义的**完整指标体系**进行统计分析：

- 结果指标（策略好不好，如增量 GMV、增量用户数）
- 过程指标（为什么好/不好，如转化率、点击率）
- 外部环境指标（供需、竞争等，如价格竞争力）

不是只算某一类指标，而是把明细数据按指标体系全量统计。

## 三、计算方法

根据数据条件选择：

| 条件 | 方法 | 说明 |
|------|------|------|
| 有 AB 实验 | 实验组 vs 对照组 | 最准确的增量计算 |
| 无实验 | 同比/环比 | 用户指定或根据数据条件判断 |
| 无实验 + 需要因果推断 | DID、断点回归等 | 准实验方法估算增量 |

具体使用什么方法由用户指定，或根据数据条件判断。

## 四、输入

- 明细数据（数据库查询结果、Excel/CSV）
- 指标体系定义（来自业务背景对齐）
- 策略明细信息（哪个策略在什么时间执行的）

## 五、输出

策略库格式的完整效果数据，可直接写入策略库。

## 六、触发时机

- 周期性执行（如每周）
- 用户要求做策略优化时触发（先更新效果数据，再做优化）

## 七、在体系中的位置

```
明细数据 → 效果评估 skill → 策略库格式效果数据 → 写入策略库
```

效果评估是策略库的**数据供给者**——策略库管理策略信息，效果评估负责把原始数据加工成策略库能用的格式。
