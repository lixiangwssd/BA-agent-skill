# 策略库 Skill — 讨论记录

## 一、策略库的两层结构

### 第一层：策略框架（定义"策略长什么样"）

策略有哪些要素维度，每个要素的可选值空间。

投放场景示例：

| 要素维度 | 可选值示例 |
|---------|-----------|
| 人（用户） | 新客、老客、高价值、低频、沉默用户… |
| 时间 | 早高峰、午间、晚间、工作日、周末… |
| 渠道 | 信息流、搜索、push、短信、开屏… |
| 供给（投放内容） | 素材A、素材B、落地页X、落地页Y… |
| 补贴（券/优惠力度） | 5元券、满30减5、免配送费… |

策略框架需要先和用户沟通对齐确认，然后才能进入数据收集阶段。

### 第二层：策略效果（记录"策略用过效果如何"）

| 组成部分 | 说明 |
|---------|------|
| 指标体系 | 结果指标（策略好不好）+ 过程指标（为什么好/不好）+ 外部环境指标（供需、竞争等） |
| 策略明细 | 具体的要素组合是什么（人群×时间×渠道×供给×补贴的具体值） |
| 执行信息 | 什么时间、什么条件下执行的策略 |
| 策略效果 | 各指标的实际数据表现 |

关键认知：**外部环境本质也是指标体系**，不是自由文本描述。比如"竞争"用"价格竞争力"指标和具体值来体现，与其他指标统一结构化记录。

## 二、冷启动机制

- 从历史数据中按策略地图结构提取填充
- 数据不全没关系，有多少填多少
- 尽量不用口述；即使用户口述，也必须最终落成策略地图的结构化格式

## 三、持续迭代更新

- skill 被触发后，执行策略沉淀动作
- 触发条件：可以是用户主动要求，也可以是定期执行（具体机制后续再定）
- 核心是"触发→沉淀"，不预设固定的触发路径

## 四、策略拟合能力

对策略库中未覆盖的策略空间，基于相邻已知策略进行效果拟合。

### 拟合逻辑

- 基于已有数据点进行插值/外推
- 利用策略要素值的连续性进行推断
- 示例：已知20-25岁和30-35岁用户效果 → 拟合出25-30岁用户的预估效果

### 拟合输出

- 拟合效果值
- 置信度标注（高/中/低）
- 拟合依据说明（基于哪些相邻策略推断）

### 验证替换

当拟合策略被实际执行后：
- 用真实数据替换拟合预估值
- 置信度从"拟合"升级为"实测"
- 记录拟合值与实测值的偏差，用于优化后续拟合精度

## 五、策略互斥关系

策略库需要和用户明确策略之间的互斥关系，作为下游策略优化 skill 的硬约束输入。

### 互斥示例

- 同一用户不能同时命中两个补贴策略（人群有重叠时只能选一个）
- 同一时段同一渠道只能投一套素材
- 同一商圈不能同时上线两个竞争性活动

### 处理方式

- 在策略框架对齐阶段，主动和用户确认互斥规则
- 结构化记录（策略A 与 策略B 互斥，原因：人群重叠）
- 下游 skill 读取互斥关系作为约束条件

## 六、策略库 Skill 的核心能力总结

1. **对齐策略框架**：和用户沟通确认策略的要素维度和值空间
2. **明确互斥关系**：和用户确认策略之间的互斥约束
3. **冷启动填充**：从历史数据中提取策略条目，按策略地图结构填入
4. **增量沉淀**：被触发时，将新策略及其效果结构化写入策略库
5. **策略拟合**：对未覆盖的策略空间进行相邻插值，标注"拟合值+置信度"
6. **验证替换**：实际执行后，用真实数据替换拟合预估，记录偏差

## 五、关键原则

- 策略框架的建立需要和用户对齐确认（同"业务背景对齐"一样的铁律）
- 所有内容创建和修改必须经用户确认，不能自行编造或擅自修改
