# 策略挖掘 Skill — 讨论记录

## 一、核心定位

通过大量数据和信息，探索挖掘更有效的新策略方案。本质是策略库的**扩展器**——策略优化在已有空间内选最优，策略挖掘负责扩大这个空间。

## 二、与策略优化的边界

| | 策略优化 | 策略挖掘 |
|---|---------|---------|
| 范围 | 已有策略空间内（框架已定义的要素×值） | 当前策略框架之外的新发现 |
| 输入 | 策略库 + 目标 + 约束 | 更广泛的原始数据 + 非结构化信息 |
| 输出 | 最优策略组合方案 | 新策略建议 + 框架调整建议 |

## 三、挖掘方向

| 类型 | 说明 | 示例 |
|------|------|------|
| A. 新的要素值 | 发现从未使用过的要素值有潜力 | "凌晨1-3点"从未投过，但该时段有高转化用户 |
| B. 新的细分方式 | 发现更有效的分群维度 | 按"用户距商圈距离"分群比按"年龄"分群效果更好 |
| C. 要素组合规律 | 发现特定组合效果突出但未被利用 | "下雨天×写字楼人群×午餐时段"效果特别好 |
| D. 跨场景借鉴 | 从其他业务场景迁移策略模式 | 外卖补贴的某个模式可迁移到到店投放 |
| E. 反常识发现 | 挑战现有业务假设 | 对高价值用户给小额补贴反而转化率更高 |
| F. 时序规律 | 策略效果随时间的变化模式 | 某策略连续投3天后效果骤降；注册后第7天是最佳触达时机 |
| G. 竞争响应模式 | 竞对动作与策略效果的关联 | 竞对降价时，加大品质类素材投放效果反而更好 |
| H. 策略缺口 | 漏斗中无策略覆盖但可干预的环节 | 某环节流失严重但从未有策略覆盖 |
| I. 策略衰减预警 | 在用策略效果正在下滑 | 素材疲劳、人群饱和，需提前准备替代方案 |
| J. 供给侧机会 | 供给充足但投放未覆盖 | 某类优质商家曝光远低于其转化潜力 |
| K. 人群迁移/生命周期 | 用户不同阶段策略敏感度变化 | 新客对红包敏感，3个月后对品质敏感 |
| L. 外部信号关联 | 外部数据与策略效果的关联 | 天气、节假日、热点事件与效果的相关性 |
| M. 组合协同/抵消效应 | 策略组合的非线性效果 | 两个单独一般的策略同时上线效果放大 |
| N. 沉默维度激活 | 从未被差异化但有区分度的维度 | 新老设备、网络类型等从未用于策略差异化 |

## 四、工作方式（三种并用）

### 4.1 基于规则/模板的扫描

- 预设一系列挖掘模板（如"找未覆盖的要素值"、"找效果下滑的策略"、"找高方差维度"）
- 对数据逐一跑这些模板，系统性地检查各挖掘方向

### 4.2 开放式数据探索

- LLM 直接阅读数据，基于业务理解自主探索
- 类似资深分析师拿到数据后"翻一翻看看有什么"

### 4.3 假设驱动

- 先基于业务逻辑生成一批假设（如"下雨天效果可能不同"）
- 然后逐一用数据验证

## 五、输入

### 结构化数据
- 来源：数据库、Excel/CSV
- 通常是明细数据（比策略库的汇总粒度更细）
- 例如：用户行为明细、订单流水、投放日志等

### 非结构化数据
- 来源：用户提供的文档、信息
- 例如：行业报告、竞品分析、业务方经验总结、会议纪要等

## 六、输出

经用户确认后：
1. **新策略建议** → 写入策略库（新增策略条目）
2. **策略框架调整建议** → 更新策略框架（如新增要素维度、新增要素值）

所有输出必须经用户确认，不能自行写入策略库。

## 七、关键原则

- 挖掘发现必须有数据支撑，不能凭空推测
- 所有建议经用户确认后才更新策略库/策略框架
- 三种工作方式互补：规则扫描保证覆盖面，开放探索保证发现深度，假设验证保证严谨性
