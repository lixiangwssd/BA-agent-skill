 1. 测试用例：设计多个用例，包含原始数据、汇总效率数据等，要能够体现多维度计算钱效
  2. 工作流程上：
  - 和用户确认业务信息（尽量有，但不是必须都收集到，如果没有则agent来输出并给用户确认是否需要修改）：目标是什么指标与目标预期值（规模、效率，尤其是指标具体是什么比如规模看GMV还是订单量还是用户量；效率看ROI还是CAC等），有什么约束（比如新客补贴率不能低于多少等），系统能力（策略允许在哪些维度上设计）
  - 基于业务信息，进行数据分析：针对数据源，统计计算整体的目标指标（包括规模、效率等）、基于数据提供的维度，分别计算单一维度结果、多维度交叉结果。注意多维度交叉容易维度爆炸，保留维度交叉后规模指标至少达到总规模的5%以上，不要过度细化，导致钱效结论不可用。统计方法上，如果有实验组和对照组数据，则通过AB评估增量。如果没有则尽量用DID的方式，选择合理基期（无策略）对比策略效果
  - 基于数据分析，进行策略建议：通过明确当前阶段的总钱效，和用户的目标、约束，判断哪些维度钱效更好，建议投入更多资源。但需要关注随着资源规模提升，效率会下降，所以一般资源在维度分配上，建议一次增长不超过30%。比如新客补贴率原来为10%，建议增加后不要超过13%
  - 基于策略建议，预估策略效果：策略调整后，基于新版策略，预估总体效果
  - 输出结论：按照金字塔原理结论先行，包括当前策略整体效果，建议策略调整方案，策略预估调整后效果；然后展开策略明细，和策略调整原因
  3. 测评上：
  - 视角全面性，是否关注了业务实际运行中的目标和约束条件；
  - 数据准确性，所有数据处理是否足够准确，不允许任何错误；
  - 策略合理性，是否找到了最优策略组合，策略效果预计提升是否合理（不应该过高或过低）
