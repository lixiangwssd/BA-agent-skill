# 策略优化Skill

## 核心能力

本skill帮助企业在补贴、投放等场景下进行策略优化，通过数据驱动的方式找到最优策略组合，提高资源使用效率。适用于O2O、共享经济、电商等平台的补贴策略优化和广告投放预算分配。

## 工作流程

### 第一步：业务信息确认

**主动与用户确认以下业务信息（尽量收集，如无法收集则基于数据推断并请用户确认）：**

1. **优化目标**：希望提升什么指标？
   - 规模指标：GMV、订单量、用户量（含新用户/老用户）、日活等
   - 效率指标：**增量ROI**、**增量CAC**、LTV、Coupon ROI等
   - 预期目标值是多少？（如：增量ROI从2提升到3、GMV提升20%）

2. **约束条件**：有哪些业务限制？
   - 新客补贴率不能低于多少？
   - 单用户补贴上限是多少？
   - 预算总额上限是多少？
   - 特定渠道/用户群必须投放的最低比例？

3. **系统能力**：策略可以在哪些维度上设计？
   - 用户维度：新老用户、用户分层、活跃度等
   - 渠道维度：各投放渠道
   - 时间维度：工作日/周末、节日/平日
   - 产品维度：不同品类/SKU

**如果用户未提供明确信息，应基于数据推断并明确告知用户假设前提。**

### 第二步：数据分析

#### 2.1 计算整体目标指标（增量概念）

**重要：效率指标必须使用增量计算，而非简单比值**

**增量ROI计算公式：**
```
增量ROI = (实验组人均GMV - 对照组人均GMV) / (实验组人均补贴 - 对照组人均补贴)
```

**增量CAC计算公式：**
```
增量CAC = (实验组人均补贴 - 对照组人均补贴) / (实验组新客数 - 对照组新客数)
       = 人均补贴增量 / 新客增量
```

**举例说明：**
- 实验组：人均GMV=100元，人均补贴=20元，新客数=100人
- 对照组：人均GMV=60元，人均补贴=10元，新客数=70人
- 增量GMV = 100 - 60 = 40元
- 增量补贴 = 20 - 10 = 10元
- **增量ROI = 40 / 10 = 4.0**（表示每投入1元增量补贴，带来4元增量GMV）

基于提供的数据，计算：
- **整体规模指标**：总GMV、总订单量、总用户数等
- **整体效率指标**：**增量ROI**、**增量CAC**、平均补贴率等

#### 2.2 单维度分析

按单一维度分组统计：
- 各维度下的规模指标（GMV、订单量）
- 各维度下的**增量效率指标**（增量ROI vs 总量ROI，增量CAC）
- 各维度占用资源比例

#### 2.3 多维度交叉分析

**注意：避免维度爆炸，控制交叉维度数量**
- 选择规模占比≥5%的维度组合进行重点分析
- 记录各组合的规模、**增量效率**指标
- 识别高效和低效的维度组合

#### 2.4 因果效果评估（如有数据）

**AB评估**：如有实验组/对照组数据，计算增量：
- 实验组vs对照组的效率差异
- **必须使用增量指标**（增量ROI、增量CAC）
- 统计显著性评估

**DID（双重差分）**：如无实验组，选择合理基期：
- 基期（无策略/低策略）vs 报告期（有策略/高策略）
- 控制时间趋势，排除自然增长
- 估算策略的真实**增量**效果

### 第三步：策略约束检查（必须在上一步之前完成）

**在生成任何策略建议之前，必须先检查所有约束是否满足：**

1. **逐条检查约束**：
   - 新客补贴率 ≥ 最低要求（如8%）
   - 单用户补贴 ≤ 上限（如20元）
   - 总预算 ≤ 上限
   - 调整幅度 ≤ 30%

2. **如果当前策略已违反约束**：必须先指出问题，再给出满足约束的调整方案

3. **约束冲突时**：如果多个约束无法同时满足，必须与用户确认优先级

### 第四步：策略建议

#### 4.1 确定当前钱效

基于数据分析结果：
- 当前**增量ROI**/**增量CAC**是多少？
- 各维度的钱效对比（哪个维度增量ROI高，哪个低）
- 当前策略的问题点在哪里？

#### 4.2 策略生成原则

**核心原则：资源增量不超过30%**
- 如某维度补贴率原为10%，建议增加后不超过13%
- 如某渠道预算原为X，建议增加后不超过X×1.3

**策略方向：**
- **增量ROI高的维度**：建议增加资源投入（但控制增量≤30%）
- **增量ROI低或为负的维度**：建议减少资源投入
- 无效/负效维度：建议停止投放

### 第五步：策略效果预估

基于新策略预估：
- 总体规模指标变化（提升%）
- 总体**增量效率**指标变化（增量ROI提升/增量CAC降低）
- 是否满足用户目标

## 关键原则

1. **增量概念优先**：效率指标必须使用增量ROI、增量CAC，而非简单总量ROI
2. **数据准确性**：所有数据计算必须准确，不允许任何错误，每一步计算需可追溯
3. **结论先行且概述精炼**：核心结论不超过300字，放在报告最开头
4. **约束优先**（重要！）：策略建议必须满足用户的约束条件，且约束检查必须先于策略生成
5. **增量可控**：单次策略调整资源增量不超过30%
6. **增量合理**：策略效果预估不应过高或过低，基于数据分析实事求是
7. **维度控制**：多维度交叉分析时，保留规模占比≥5%的组合
8. **数据验证**：在分析之前必须对CSV原始数据进行验证计算

## 测评标准（内部指引，供模型自我检验）

1. **视角全面性**：是否关注了业务实际运行中的目标和约束条件
2. **数据准确性**：所有数据处理是否足够准确，所有计算必须可验证
3. **策略合理性**：是否找到了增量ROI最优策略组合，策略效果预计提升是否合理
4. **增量指标正确性**：效率指标是否使用增量概念，计算公式是否正确
