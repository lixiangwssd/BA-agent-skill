# AB测试策略分析报告

## 一、数据概览

### 1.1 数据结构
- **对照组（低补贴策略）**：3个月数据，2个渠道（APP/小程序）× 2个用户类型（新客/老客）
- **实验组（高补贴策略）**：同上结构

### 1.2 核心指标定义
| 指标 | 计算公式 | 说明 |
|------|----------|------|
| 补贴率 | 补贴金额 / GMV | 衡量补贴力度 |
| 人均补贴 | 补贴金额 / 用户数 | 单位用户补贴成本 |
| CAC | 补贴金额 / 新客数 | 新客获取成本 |
| ROI | GMV / 补贴金额 | 补贴投入产出比 |

## 四、增量分析（实验组 - 对照组）

### 4.1 核心增量指标

| 指标 | 对照组 | 实验组 | 增量 | 增量率 |
|------|--------|--------|------|--------|
| 总GMV | 480,000 | 654,780 | **+174,780** | **+36.4%** |
| 总补贴 | 75,000 | 97,440 | **+22,440** | **+29.9%** |
| 新客数 | 222 | 347 | **+125** | **+56.3%** |
| 订单量 | 11,220 | 14,637 | **+3,417** | **+30.5%** |
| 整体ROI | 6.40 | 6.72 | **+0.32** | **+5.0%** |
| 新客补贴率 | 25.0% | 20.0% | **-5.0pp** | -20.0% |
| 新客CAC | 135.1元 | 119.8元 | **-15.3元** | **-11.3%** |

### 4.2 增量ROI计算

**增量ROI = (实验组GMV - 对照组GMV) / (实验组补贴 - 对照组补贴)**
- 增量GMV = 654,780 - 480,000 = 174,780元
- 增量补贴 = 97,440 - 75,000 = 22,440元
- **增量ROI = 174,780 / 22,440 = 7.79**

### 4.3 增量CAC计算

**增量CAC = 增量补贴 / 增量新客数**
- 增量补贴中用于新客：41,580 - 30,000 = 11,580元
- 增量新客数：347 - 222 = 125人
- **增量CAC = 11,580 / 125 = 92.6元/人**

## 七、策略建议

### 7.1 整体结论

| 评估维度 | 结果 | 是否达标 |
|----------|------|----------|
| GMV增长 | +36.4%（目标20%） | ✅ 超额完成 |
| 增量ROI | 7.79（目标≥3） | ✅ 达标 |
| 新客补贴率 | 20%（目标≥8%） | ✅ 达标 |
| 单用户补贴上限 | 119.6~150元（上限20元） | ❌ 不满足 |

### 7.2 问题诊断

**高补贴策略带来了显著的GMV增长和ROI提升，但存在一个核心问题**：
- 新客人均补贴（119.6~150元）远超单次补贴上限20元
- 这可能是数据口径问题（如果是累计补贴，则需拆分）
- 建议确认补贴统计方式，是否为订单级补贴

### 7.3 优化建议

#### 建议1：调整新客补贴策略（优先级：高）

**现状问题**：新客人均补贴120元以上，远超20元上限

**优化方向**：
- 将新客补贴从"按人头"改为"按订单"补贴
- 单次补贴上限20元，确保合规
- 通过提高新客转化率（从132人到198人，+50%）来提升整体效果

**预估效果**：
- 新客补贴率可降至15%左右
- 在保持ROI≥3的前提下，进一步提升GMV

#### 建议2：加大小程序渠道投入（优先级：中）

**现状**：小程序渠道增量ROI表现更优（6.76 vs APP 6.69）

**优化方向**：
- 将小程序预算占比从33%提升至40%
- 小程序新客增长65.6%，高于APP的50%

**预估效果**：
- 预计整体GMV再提升5~8%

#### 建议3：优化老客补贴策略（优先级：中）

**现状**：老客补贴率12.5%，ROI贡献稳定

**优化方向**：
- 保持老客补贴率在10~12%区间
- 通过会员体系、复购激励等方式提升老客GMV贡献

**预估效果**：
- 老客GMV占比约68%，稳定基本盘
