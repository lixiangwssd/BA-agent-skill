# 指标异动归因分析方法论

## 适用范围

当需要分析公司一个或多个指标的异常变化时，使用本参考资料。目标不只是检测变化，而是解释每个维度贡献了多少变化，以及什么业务机制可能导致这些变化。

## 1. 判断指标是否真的发生异动

先把指标定义说清楚：

- 指标公式：分子、分母、筛选条件、去重逻辑、归因窗口和聚合粒度。
- 观察周期：当前周期和对比周期。
- 变化方向：上升、下降或结构性变化。
- 影响规模：绝对变化、相对变化，以及可量化的业务价值。

将变化归类：

- `真实异动`：变化明显大于正常波动，并通过数据质量检查。
- `正常波动`：变化处于历史波动范围内，或可由周期性、季节性解释。
- `数据质量问题`：很可能由埋点、ETL、筛选条件、延迟数据、去重、schema 或指标口径变化导致。
- `暂无法判断`：数据不足，或对比周期本身不可靠。

常用检查：

- 历史波动：有数据时，对比过去 7、14、28 或 56 个周期。
- 季节性基线：对比相同星期、相同活动阶段、相同节假日窗口或相同生命周期阶段。
- 样本量：确认分子和分母是否足够大，可以支撑稳定判断。
- 数据新鲜度：确认延迟到达数据是否可能改变结论。
- 指标链路：检查来源表、看板筛选条件、归因逻辑、机器人过滤和版本变化。

## 2. 贡献度拆解

基础问题是：`哪些分段解释了总指标变化？`

对加总型指标：

```text
分段变化量 = 当前分段值 - 基准分段值
总变化量 = 当前总值 - 基准总值
贡献占比 = 分段变化量 / 总变化量
```

解释方式：

- 正贡献表示该分段推动总指标朝总变化方向移动。
- 负贡献表示该分段抵消了总指标变化。
- 当其他分段存在抵消时，某个分段贡献占比可能超过 100%。

对比率型指标：

```text
指标 = 分子 / 分母
```

数据允许时，至少拆分三类效应：

- 分子效应：分子发生变化，而分母结构大致稳定。
- 分母效应：分母规模变化，导致稀释或集中。
- 结构效应：流量、用户、产品或渠道构成发生变化。

对漏斗型指标：

- 将最终结果拆成流量规模、各阶段转化率和结构变化。
- 判断异动是从上游开始，还是从下游开始。
- 按分段比较各阶段变化。

对多维分析：

1. 从高层维度开始。
2. 按对总变化量的贡献度排序。
3. 对贡献最大的维度继续下钻。
4. 当新增维度不再解释显著变化，或样本量过小时停止。

避免常见误区：

- 不要只按百分比增长排序；小分段容易显得变化很剧烈。
- 不要忽略抵消项；抵消项常常能解释为什么总指标变化小于预期。
- 不要在未检查重叠关系时，把相关维度当成独立原因。
- 不要过早合并所有维度；根因可能藏在某个细分分段里。

## 3. 业务归因

把数据驱动因素转化为业务假设。强归因需要连接：

```text
业务事件或机制 -> 受影响分段 -> 指标组成部分 -> 观察到的总变化
```

常见原因类型：

- 产品：功能上线、功能缺陷、体验变化、推荐策略变化、定价或包装变化。
- 增长和营销：活动、渠道结构、投放变化、素材疲劳、归因逻辑、受众质量。
- 运营：供给、库存、履约、审核、客服、SLA、排班。
- 用户行为：cohort 老化、留存变化、频次变化、偏好变化、流失、召回。
- 市场和日历：节假日、天气、竞品动作、宏观变化、开学或工作周期。
- 数据和埋点：埋点发布、ETL 延迟、schema 变化、看板筛选条件、重复事件。

对每个假设记录：

- 支持证据：哪些具体数据模式支持该假设。
- 反向证据：如果该假设成立，理论上应该出现但实际没有出现的现象。
- 缺失验证：还需要哪些数据来验证或证伪。
- 置信度：高、中、低。

## 5. 最终叙述模板

使用简洁的业务语言：

```text
该指标看起来属于 [真实异动 / 正常波动 / 数据质量问题 / 暂无法判断]。
指标从 [基准值] 变化到 [当前值]，在 [周期] 内变化了 [变化量]。
最大驱动因素是 [维度：分段]，贡献了总变化的 [x%]。
最可能的业务解释是 [假设]，因为 [证据]。
不过，[不确定性/反向证据]。下一步建议验证 [具体检查项]。
```
