# 指标异动分析报告

## 一、概况

| 项目 | 数值 |
|------|------|
| 分析指标 | 注册转化率（公式：提交成功数 / 落地页访问 UV） |
| 对比方式 | 本月 vs 上月（月环比） |
| 指标变化 | -1.1 个百分点（3.2% → 2.1%） |
| 用户损失 | 约 5,080 人（16000 → 10920） |
| 异动判定 | 真实异动 |

## 二、异动验证

- 转化率从 3.2% 降至 2.1%，相对下降 34.4%，属于显著变化
- 落地页 UV 本月反而增长 4%（50 万 → 52 万），排除流量萎缩因素
- 各步骤转化率全面下降，非单点问题
- **判定：真实异动，漏斗内部多环节均有恶化**

## 三、贡献度拆解

### 3.1 各步骤转化率变化

| 步骤 | 上月转化率 | 本月转化率 | 变化 | 上游UV基数 | 该步绝对损失 |
|------|-----------|-----------|------|-----------|------------|
| 点击注册按钮 | 25.0% | 21.0% | -4.0pp | 520,000 | -20,800 UV |
| 填写表单 | 50.0% | 45.0% | -5.0pp | 109,200 | -5,460 UV |
| 提交成功 | 25.6% | 22.2% | -3.4pp | 49,140 | -1,671 UV |

### 3.2 贡献度排序

| 优先级 | 步骤 | 绝对损失 | 贡献权重 | 说明 |
|--------|------|---------|---------|------|
| P0 | 点击注册按钮 | -20,800 UV | 74.5% | **最大漏点，漏斗顶部影响全局** |
| P1 | 填写表单 | -5,460 UV | 19.5% | 次要漏点 |
| P2 | 提交成功 | -1,671 UV | 6.0% | 相对影响较小 |

> 「点击注册按钮」步骤虽然转化率降幅（-4pp）不是最大的，但因为处于漏斗顶部、基数最大（52万UV），导致绝对损失最大，是本次异动的核心漏点。

### 3.3 流量变化 vs 转化率变化

- 流量侧：落地页 UV 增长 4%（+2万），流量不是问题
- 转化率侧：三步转化率全面下降，是纯粹的转化效率问题
- **结论：本次异动 100% 由转化率下降驱动，非流量问题**

## 四、根因分析

### 4.1 主要驱动因素

**P0：点击注册按钮转化率下降（25% → 21%，-4pp）**
- What：落地页到点击注册按钮的转化率大幅下降
- How much：损失约 20,800 个进入下一步的 UV
- Why（假设）：
  - 落地页改版（按钮位置、颜色、文案变化）
  - 页面加载速度变慢
  - 流量质量变化（新增渠道带来低意向用户）
  - 竞品拦截（用户比价后流失）

**P1：填写表单转化率下降（50% → 45%，-5pp）**
- What：点击注册后到填写完表单的流失增加
- How much：损失约 5,460 UV
- Why（假设）：
  - 表单字段增加或变复杂
  - 验证码体验变差（短信延迟、图形验证码难度）
  - 手机端适配问题

### 4.2 假设验证

| 假设 | 支持证据 | 反向证据 | 置信度 |
|------|---------|---------|--------|
| 落地页改版导致按钮点击下降 | 转化率断崖式下降 4pp | 待验证是否有版本发布 | 高 |
| 流量质量下降 | 流量增长 4% 但转化全面下降 | 需看分渠道转化差异 | 中 |
| 表单体验变差 | 填写步骤降 5pp 是最大百分点降幅 | 待验证表单变更记录 | 中 |
| 系统/性能问题 | 三步全面下降符合系统性问题特征 | 待验证错误日志和加载时长 | 中 |

### 4.3 已排除项

- **流量不足**：落地页 UV 反而增长 4%
- **季节性因素**：需确认，但 -34% 的降幅过大，不太可能是正常季节波动

## 五、结论

注册转化率下降 1.1pp（-34%）属于显著异动，核心漏点在「点击注册按钮」步骤（贡献 74.5% 损失）。三步转化率全面下降，可能存在系统性原因（如落地页改版或性能退化），也可能是流量结构变化（新增低质量流量）导致。

## 六、行动建议

1. **【P0/产品团队】排查落地页近期改动**：是否有 A/B 实验上线、按钮样式/位置变化、文案调整，如有则回滚验证
2. **【P0/技术团队】检查页面性能**：落地页加载时长、首屏渲染时间，对比上月基线
3. **【P1/产品团队】检查表单和验证码环节**：表单字段是否增加、短信验证码到达率和延迟
4. **【P1/增长团队】分渠道看转化率**：区分自然流量和付费流量，判断是否是新增渠道拉低整体

## 七、数据局限

- 缺少分渠道/分端（iOS/Android/H5）的漏斗数据，无法判断是否为特定端问题
- 缺少页面性能数据（加载时长、JS 错误率）
- 缺少版本发布记录，无法直接关联产品变更
- 建议补充：分端漏斗数据、近期发版记录、页面性能监控、短信到达率
