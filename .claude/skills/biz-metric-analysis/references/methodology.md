### 1.0 澄清流程

在开始分析前，如果关键信息不足，必须主动追问：

| 缺失信息 | 追问内容 |
|---------|---------|
| 指标不明确 | "请问是哪个指标？" |
| 时间段不明确 | "对比哪个时间段？" |
| 数据未提供 | "有数据文件或表格可以提供吗？" |
| 用户只说"波动" | "请问是哪个指标？最近什么时候开始波动？" |

**原则**：不能跳步，不能在信息不足时强行给出一个可能不准确的结论。

## 1. 异动验证方法

### 1.1 指标定义确认

分析前先明确：
- 指标公式：分子、分母、筛选条件、去重逻辑、归因窗口、聚合粒度
- 观察周期：当前周期和对比周期
- 变化方向和规模：绝对变化、相对变化、业务影响金额

### 1.2 统计检验

**Z-score 法**（适用于有历史数据）：
```
z = (当前值 - 历史均值) / 历史标准差
|z| > 2 → 显著异动（约 5% 误报率）
|z| > 3 → 极端异动（约 0.3% 误报率）
```

**IQR 法**（适用于分布偏斜的指标）：
```
下界 = Q1 - 1.5 × IQR
上界 = Q3 + 1.5 × IQR
超出上下界 → 异常值
```

### 1.3 排除干扰因素

按顺序检查：
1. 季节性/日历效应：同星期对比、同节假日窗口、同活动阶段
2. 分母变化：分母规模突变可能导致比率指标假性波动
3. 样本量：分子分母是否足够大，小样本不宜下结论
4. 数据新鲜度：T+1 数据延迟可能导致近期数据偏低
5. 指标链路：埋点发布、ETL 逻辑变更、看板筛选条件变化、去重逻辑变化

## 2. 贡献度拆解公式

### 2.1 加总型指标

```
分段变化量 = 当前分段值 - 基准分段值
总变化量 = 当前总值 - 基准总值
贡献占比 = 分段变化量 / 总变化量
```

注意：
- 正贡献 = 该分段推动总指标朝总变化方向移动
- 负贡献 = 该分段抵消了总指标变化
- 当存在抵消时，某分段贡献占比可能超过 100%

### 2.2 乘法模型（价量分析）

当 指标 = A × B：
```
总变化 = A当前×B当前 - A基准×B基准
      = 量变效应 + 价变效应 + 交叉效应

量变效应 = (A当前 - A基准) × B基准
价变效应 = (B当前 - B基准) × A基准
交叉效应 = (A当前 - A基准) × (B当前 - B基准)
```

三因素分解（指标 = A × B × C）：
```
总变化 = A效应 + B效应 + C效应 + 交叉效应
A效应 = (A当前 - A基准) × B基准 × C基准
B效应 = A基准 × (B当前 - B基准) × C基准
C效应 = A基准 × B基准 × (C当前 - C基准)
交叉效应 = 总变化 - A效应 - B效应 - C效应
```

### 2.3 比率型指标

rate = 分子 / 分母

三类效应：
- **分子效应**：分子变化，分母结构大致稳定
- **分母效应**：分母规模变化导致稀释或集中
- **结构效应**：子群占比变化（Simpson 悖论风险）

结构效应检测：
```
整体转化率变化 = Σ(分段转化率变化 × 分段权重) + Σ(分段权重变化 × 分段转化率)
                = 组内效应 + 结构效应
```

### 2.4 漏斗型指标

拆解维度：
- 流量规模变化（入口 UV）
- 各阶段转化率变化
- 流量结构变化（不同来源的转化率天然不同）

判断起点：从上游到下游逐步检查，找到第一个显著下降的环节。

### 2.5 多维下钻

#### 2.5.1 多维并行拆解流程

当数据包含多个维度（渠道、品类、地区等）时，按以下步骤分析：

```
Step 1: 并行拆解所有维度
       计算每个维度对总变化的贡献度

Step 2: 找出贡献度最大的维度
       按贡献度排序，确定下钻优先级

Step 3: 对贡献最大的维度做交叉验证
       维度1 × 维度2 交叉分析（如渠道×地区）

Step 4: 定位最大拖累组合
       结合已知背景归因
```

#### 2.5.2 交叉维度分析示例

```python
# 并行计算所有维度贡献度
dimensions = ['渠道', '品类', '地区']
contributions = {}
for dim in dimensions:
    dim_change = df.groupby(dim)['变化量'].sum()
    contributions[dim] = dim_change.abs().sum()

# 找出贡献最大的维度
top_dims = sorted(contributions.items(), key=lambda x: x[1], reverse=True)[:2]
# 输出：[('渠道', 120), ('地区', 120)]

# 对贡献最大的两个维度做交叉分析
cross = df.groupby([top_dims[0][0], top_dims[1][0]])['变化量'].sum()
cross_sorted = cross.sort_values(ascending=True)
# 找到最大拖累组合
top_combo = cross_sorted.index[0]
```

#### 2.5.3 多维下钻决策原则

| 情况 | 操作 |
|------|------|
| 维度 A 贡献 >> 其他维度 | 直接对 A 下钻 |
| 多维度贡献接近 | 多维度交叉验证 |
| 交叉维度后贡献集中 | 继续下钻直到定位根因 |
| 下钻后样本量不足 | 停止下钻，汇总结论 |

## 3. 业务归因框架

### 3.1 从数据到假设

强归因需要连接：
```
业务事件/机制 → 受影响分段 → 指标组成部分 → 观察到的总变化
```

### 3.2 常见原因分类

| 类别 | 典型原因 | 交叉验证指标 |
|------|---------|-------------|
| 产品 | 功能上线、bug、体验改版、推荐策略 | 页面停留时长、退出率、功能使用率 |
| 增长/投放 | 预算变化、素材疲劳、渠道调整、归因变化 | 消耗、曝光、CPM、CTR、CPA |
| 运营 | 活动上下线、补贴策略、供给变化、定价 | 券核销率、活动GMV占比、商家数 |
| 用户 | 新老客占比、cohort 老化、竞品分流 | 新客占比、7日留存、频次 |
| 外部 | 节假日、天气、宏观、政策 | 全行业数据、竞品公开数据 |
| 数据 | 埋点变更、ETL 延迟、口径变化 | 数据量突变、空值率、延迟到达 |

### 3.3 假设置信度评估

| 置信度 | 标准 |
|--------|------|
| 高 | 有直接数据证实 + 无明显反向证据 + 时间线吻合 |
| 中 | 有间接证据支持，但存在其他可能解释 |
| 低 | 基于业务经验推断，缺乏数据验证 |

### 3.4 矛盾现象处理

当出现矛盾指标时（如 DAU↓ 但留存↑），处理框架：

```
1. 识别矛盾：列出所有矛盾指标对
2. 提出假设：建立能同时解释矛盾双方的假设
3. 设计验证：明确需要什么数据来验证假设
4. 执行验证：对比不同群体的数据
5. 得出结论：确定是真实异动还是口径变化
```

**示例：DAU 下降但留存上升**
```
假设：版本更新导致体验好的老用户留下，体验差的新用户流失

验证步骤：
1. 分群对比：新用户 vs 老用户的 DAU 变化
2. 版本对比：更新前 vs 更新后的各指标趋势
3. 渠道对比：各渠道的新增用户转化率差异

结论判定：
- 若老用户留存↑ + 新用户 DAU↓ → 支持假设
- 若各群体同步变化 → 可能是数据口径问题
```

## 4. 常见误区

1. **只看增长率不看基数**：小分段 +200% 可能只贡献总变化的 2%
2. **忽略抵消项**：正负贡献互相抵消时，单看净值会低估实际波动
3. **混淆相关性和因果性**：时间上巧合 ≠ 因果关系，需要机制解释
4. **过度归因单一因素**：多因素同时作用时不要强行归结为一个原因
5. **忽视 Simpson 悖论**：整体下降可能是结构变化而非各分段都在下降
6. **把正常波动当异动**：没有历史基线就随意判断"异常"
7. **数据质量问题当业务问题**：先排查数据链路再分析业务原因
